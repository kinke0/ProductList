# 数据审批流设计

## 概述

为信息清单中的每条数据添加审批流程，通过状态机控制数据的编辑权限和审批流转。

## 状态机

4个状态：

```
待提交 → 待审核 → 审核通过
  ↑          │          │
  └ 驳回 ←───┘          │
           驳回 ←───────┘
```

### 状态定义

| 状态 | 值 | 说明 |
|------|------|------|
| 待提交 | `待提交` | 新建数据的默认状态 |
| 待审核 | `待审核` | 编辑员已提交，等待审核员审核 |
| 审核通过 | `审核通过` | 审核员通过，数据锁定 |
| 驳回 | `驳回` | 审核员驳回，编辑员可重新编辑 |

### 状态流转

| 当前状态 | 操作 | 目标状态 | 操作角色 |
|---------|------|---------|---------|
| 待提交 | 提交 | 待审核 | 编辑员、管理员 |
| 待审核 | 通过 | 审核通过 | 审核员、管理员 |
| 待审核 | 驳回 | 驳回 | 审核员、管理员 |
| 审核通过 | 驳回 | 驳回 | 审核员、管理员 |
| 驳回 | 提交 | 待审核 | 编辑员、管理员 |

## 角色权限

### 角色定义

| 角色 | 值 | 说明 |
|------|------|------|
| 编辑员 | `editor` | 可编辑和提交数据 |
| 审核员 | `reviewer` | 可审核通过或驳回数据 |
| 管理员 | `admin` | 拥有所有权限 |

### 权限矩阵

| 状态 | 编辑员 | 审核员 | 管理员 |
|------|--------|--------|--------|
| 待提交 | 可编辑、可提交 | - | 全部权限 |
| 待审核 | 不可编辑 | 可通过、可驳回 | 全部权限 |
| 审核通过 | 不可编辑 | 可驳回 | 全部权限 |
| 驳回 | 可编辑、可提交 | - | 全部权限 |

## 前置条件

- 只有版本状态为 `draft`（编辑中）时才能变更审批状态
- 已发布版本的审批状态不可变更

## 数据变更

### User 实体

新增 `role` 字段：

- 字段：`role`
- 类型：String
- 取值：`editor`、`reviewer`、`admin`
- 默认值：`editor`

### DataEntry 实体

复用现有 `colStatus` 字段存储审批状态：

- 取值范围：`待提交`、`待审核`、`审核通过`、`驳回`
- 新建数据默认值：`待提交`

## 后端接口

### 审批操作接口

`POST /api/data/{id}/approve`

请求体：
```json
{
  "action": "submit|approve|reject"
}
```

- `submit`：待提交 → 待审核，或 驳回 → 待审核
- `approve`：待审核 → 审核通过
- `reject`：待审核 → 驳回，或 审核通过 → 驳回

权限校验：
1. 版本必须为 `draft`
2. 校验用户角色是否有权执行该操作
3. 校验当前状态是否允许该操作

### 批量审批接口

`POST /api/data/batch-approve`

请求体：
```json
{
  "entryIds": [1, 2, 3],
  "action": "submit|approve|reject"
}
```

## 前端变更

### 状态标签颜色

| 状态 | 颜色 | el-tag type |
|------|------|-------------|
| 待提交 | 蓝色 | primary |
| 待审核 | 橙色 | warning |
| 审核通过 | 绿色 | success |
| 驳回 | 红色 | danger |

### 编辑权限控制

- 根据当前用户角色 + 数据状态决定是否显示编辑/删除按钮
- `待审核` 和 `审核通过` 状态下隐藏编辑按钮（管理员除外）
- 状态列旁显示审批操作按钮（提交/通过/驳回）

### 审批操作按钮

在数据行的操作列中，根据状态和角色动态显示：

| 状态 | 编辑员可见按钮 | 审核员可见按钮 | 管理员可见按钮 |
|------|---------------|---------------|---------------|
| 待提交 | 提交 | - | 提交、编辑、删除 |
| 待审核 | - | 通过、驳回 | 通过、驳回、编辑 |
| 审核通过 | - | 驳回 | 驳回、编辑 |
| 驳回 | 提交 | - | 提交、编辑、删除 |
